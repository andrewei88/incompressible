1. LLM functions as the agent's brain, complemented by Planning, Memory, and Tool Use components
2. Task Decomposition via Chain of Thought (think step by step) and Tree of Thoughts (BFS/DFS over thought tree)
3. LLM+P uses Planning Domain Definition Language (PDDL) as interface to an external classical planner
4. ReAct integrates reasoning and acting with Thought/Action/Observation template; Reflexion adds dynamic memory + binary reward with heuristic to stop hallucinated trajectories
5. Memory types: sensory (seconds), short-term/working (~7 items, 20–30s, in-context limited by Transformer window), long-term (external vector store with fast retrieval)
6. MIPS / approximate nearest neighbors: LSH, ANNOY, HNSW, FAISS, ScaNN
7. MRKL (neuro-symbolic router with expert modules), TALM/Toolformer (fine-tune LMs to use external APIs), HuggingGPT (task planning → model selection → execution → response summarization)
8. API-Bank: 53 tools, 264 dialogues, three evaluation levels (call, retrieve, plan beyond retrieve)
9. ChemCrow: LLM-based eval said GPT-4 ≈ ChemCrow, but human expert eval showed ChemCrow outperforms — problem with LLM self-evaluation in deep domains
10. Boiko et al. chemical weapon test: 4 of 11 (36%) requests accepted, 7 of 11 rejected
11. Generative Agents (Park 2023): 25 characters, memory stream, retrieval by relevance/recency/importance, reflection, emergent social behavior
12. Challenges: finite context length, long-term planning/task decomposition hard, unreliable natural language interface
